Bem-vindo ao Mestre de Obra
Configure as integrações uma única vez e o sistema opera sozinho — backup automático, IA integrada e relatórios com um clique.
Configuração completa: 20–40 minutos
Progresso da configuração 0 de 5 concluídos
1
Hospedar o arquivo HTML
Escolha uma das 3 opções — todas gratuitas
▾
Netlify Drop ⚡ Mais rápido
Arrasta a pasta e em 30 segundos tem link. Ideal para começar agora.
1
Crie uma pasta chamada mestre-obra e coloque o arquivo HTML dentro. Renomeie para index.html.
2
Acesse app.netlify.com/drop e arraste a pasta inteira para a área cinza da página.
3
Em segundos aparece um link. Crie uma conta gratuita no Netlify para que o link seja permanente — sem conta ele expira em 24h.
4
Para atualizar o app futuramente: arraste a pasta novamente no mesmo site do Netlify — o link permanece o mesmo.
Sem criar conta, o link expira em 24 horas. Crie a conta gratuita na hora para garantir o link permanente.
GitHub Pages ★ Recomendado
Link permanente, histórico de versões e controle total. Leva ~15 minutos na primeira vez.
1
Acesse github.com e crie uma conta gratuita (se ainda não tiver).
2
Clique em New repository. Nome: mestre-obra. Marque "Public". Clique em "Create repository".
3
Na página do repositório, clique em "uploading an existing file". Arraste o arquivo HTML e renomeie para index.html. Clique em "Commit changes".
4
Vá em Settings → Pages. Em "Source" escolha "Deploy from a branch". Branch: main. Pasta: / (root). Clique em Save.
5
Aguarde ~2 minutos. O link aparece no topo da página Pages: https://seu-usuario.github.io/mestre-obra
6
Para atualizar: suba o novo arquivo HTML pelo mesmo caminho — o GitHub detecta automaticamente e republica em ~1 minuto.
O link do GitHub Pages é permanente e gratuito para sempre. É a opção mais robusta para uso contínuo.
Google Drive + drv.tw Alternativa
Usa o Drive que você já tem. Precisa de um serviço intermediário para renderizar o HTML.
1
Faça upload do arquivo HTML no Google Drive. Clique com o botão direito → "Compartilhar" → "Qualquer pessoa com o link" → copie o link.
2
O link do Drive tem o formato: drive.google.com/file/d/ID_DO_ARQUIVO/view. Copie apenas o ID_DO_ARQUIVO (a parte longa entre /d/ e /view).
3
Acesse drv.tw, cole o ID do arquivo e clique em "Get Link". O site gera um link que renderiza o HTML diretamente.
4
Copie o link gerado pelo drv.tw — esse é o link para usar no celular e compartilhar.
O drv.tw é um serviço de terceiros não controlado pela Berti. Se ficar fora do ar, o link para de funcionar. Use Netlify ou GitHub Pages para maior confiabilidade.
📱
iPhone (todas as opções): abra o link no Safari → toque em Compartilhar → "Adicionar à Tela de Início". O app vira um ícone como qualquer outro app, abre em tela cheia.
✓
Arquivo está online e acessível pelo celular
2
Criar a Planilha de Backup
Google Sheets recebe os dados automaticamente a cada 30 segundos
▾
1
Acesse sheets.google.com e crie uma planilha em branco. Dê o nome: Mestre de Obra — Sync Berti
2
Dentro da planilha: Extensões → Apps Script. Apague o código padrão que aparece.
3
Aqui no app, vá em Aba Obra → Integrações → Planilha de Sync → "Ver código do script". Copie todo o código e cole no Apps Script.
4
No Apps Script: clique em Implantar → Nova implantação → Aplicativo da Web. Executar como: "Eu". Quem acessa: "Qualquer pessoa". Clique em Implantar.
5
Autorize o acesso quando solicitado (tela "avançado" → "não seguro" → Permitir — é normal para scripts pessoais). Copie a URL gerada.
6
De volta ao app: cole a URL da planilha e a URL do script nos campos da Planilha de Sync. Clique em "Testar conexão".
Quando a conexão estiver ok, aparece o pill verde no topo com o horário do último backup. A partir daí o backup sobe automaticamente.
✓
Planilha criada e conexão testada com sucesso
3
Ativar a IA (Anthropic)
Briefings automáticos · Mensagens para cliente · Análise de obra
▾
1
Acesse console.anthropic.com e crie uma conta. Clique em API Keys → Create Key. Nomeie como mestre-obra e copie a chave (começa com sk-ant-).
2
Modo recomendado (proxy): no Apps Script de Sync, acesse Configurações do projeto → Propriedades do script. Adicione: nome ANTHROPIC_KEY, valor = sua chave. A chave fica protegida no servidor Google.
3
Alternativa (local): no app, aba Obra → Integrações → API Anthropic. Cole a chave e clique em Salvar.
O custo é muito baixo — tipicamente menos de R$ 5/mês para o uso do sistema. Há crédito gratuito inicial ao criar a conta.
✓
Chave Anthropic configurada (proxy ou local)
4
Fotos em nuvem (ImgBB)
Fotos hospedadas online — sem ocupar espaço no celular
▾
1
Acesse api.imgbb.com e crie uma conta gratuita.
2
Faça login → acesse as configurações → clique em "Get API key". Gere e copie a chave (sequência hexadecimal).
3
No app, aba Obra → Integrações → API ImgBB. Cole a chave e clique em Salvar.
Sem ImgBB, fotos ficam como base64 no navegador — ocupa muito espaço e deixa o app lento. Recomendamos configurar antes de enviar a primeira foto.
✓
Chave ImgBB configurada
5
Cadastrar a primeira obra
Dados da obra, empresa, equipes e serviços
▾
1
Clique em "+ Nova Obra" na tela de seleção. O app abre direto na aba Obra com a seção de configuração aberta.
2
Preencha: Nome da obra, Cliente, Semana 1, Período (data início e fim da semana). Esses campos são suficientes para começar a operar.
3
Na seção Empresa: preencha CNPJ, telefone, PIX e faça upload do logotipo. Esses dados aparecem em todos os relatórios.
4
Na aba Equipes: cadastre cada equipe com código (ex: EQ01), nome e responsável.
5
Na aba Serviços: cadastre os serviços com código, setor, equipe responsável e datas de início/fim.
6
Na aba Campo: marque a presença das equipes e use os botões 0%/25%/50%/75%/100% para atualizar o avanço de cada serviço.
Na sexta-feira: aba Campo → "Processar semana" → a IA gera o relatório completo com base no que foi registrado durante a semana.
✓
Primeira obra cadastrada com equipes e serviços
